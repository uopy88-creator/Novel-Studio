# Novel Studio UI Guide

이 문서는 Novel Studio의 **디자인 원칙**을 정리한다.  
새 화면을 만들 때 “예쁜가?”보다 **이 원칙을 지키는가?**를 먼저 본다.

참고 제품: **Notion**, **Obsidian**, **Linear**

---

## 한 줄 목표

> 심플하고, 오래 봐도 질리지 않는 작업실 UI

화려함보다 **집중**과 **여백**을 우선한다.

---

## 디자인 원칙

### 1. 심플하게

- 한 화면에 역할을 너무 많이 넣지 않는다
- 장식용 배지, 글로우, 과한 그림자를 피한다
- 아이콘·이모지는 필요할 때만 (사이드바 등)

### 2. 밝고 조용한 색

| 역할 | 방향 |
|------|------|
| 배경 | **흰색** |
| 구분/패널 | **매우 연한 회색** |
| 포인트 | **파란색 하나** |
| 텍스트 | 거의 검정에 가까운 잉크색 |

포인트 컬러는 “주요 액션 / 활성 메뉴”에만 쓴다.  
화면 전체가 파랗게 물들지 않게 한다.

### 3. 둥근 UI

- 버튼: 둥근 모서리 (주요 CTA는 pill에 가깝게도 가능)
- 카드: 부드러운 radius
- 입력창·모달: 같은 계열의 radius로 통일

각진 신문형 레이아웃이나 과한 네온 UI는 쓰지 않는다.

### 4. 카드 UI

- 목록 항목(작품, 챕터 등)은 카드로 묶는다
- 카드는 **얇은 보더 + 흰 배경**이 기본
- 그림자는 거의 없거나 아주 약하게

### 5. 여백을 많이

- 답답한 밀집 레이아웃을 피한다
- 섹션 사이, 카드 안 패딩을 넉넉히
- iPad에서도 터치하기 쉬운 크기 (대략 최소 44px 터치 영역)

---

## 레이아웃 구조

작품 선택 후 작업실:

```
┌──────────┬─────────────────────┐
│ Sidebar  │ Header (작품 제목)   │
│          ├─────────────────────┤
│          │ Content             │
│          │                     │
└──────────┴─────────────────────┘
```

| 영역 | 역할 |
|------|------|
| Sidebar | 메뉴 이동, 접기/펼치기 |
| Header | **현재 작품 제목만** 표시 |
| Content | 기능 화면 본문 |

첫 화면(작품 목록)에는 사이드바 작업실 레이아웃을 쓰지 않는다.

## 컴포넌트 가이드 (기존 디자인 시스템)

구현 위치: `src/components/ui/`, 토큰: `src/app/globals.css`

| 컴포넌트 | 용도 |
|----------|------|
| Button | primary / secondary / ghost / danger |
| Card | 목록·섹션 컨테이너 |
| Input | 짧은 텍스트 입력 |
| Modal | 생성·수정·삭제 확인 |

토큰 접두사 예:

- 색: `ns-canvas`, `ns-muted`, `ns-accent`, `ns-ink`
- 간격: `ns-4`, `ns-6`, `ns-8` …
- 라운드: `rounded-ns-md`, `rounded-ns-lg`

## 하지 말 것

| 피하기 | 이유 |
|--------|------|
| 보라/네온 그라데이션 테마 | 작업실 톤과 안 맞음 |
| 다크 모드 기본값 | 현재 컨셉은 밝은 작업실 |
| 히어로에 통계·배지 잔뜩 | 산만함 |
| 기능마다 다른 버튼 스타일 | 일관성 깨짐 |

## 체크리스트 (PR / 새 화면)

새 UI를 넣기 전에 스스로 확인한다.

- [ ] 흰색 + 연회색 + 포인트 블루만 썼는가?
- [ ] 카드/버튼이 둥글고 과하지 않은가?
- [ ] 여백이 충분한가?
- [ ] iPad에서 탭하기 어려운 작은 버튼이 없는가?
- [ ] Header에 작품 제목 외에 잡다한 정보를 넣지 않았는가?
