## 디자인 원칙

### 색

| 역할 | 방향 |
|------|------|
| 배경 | **흰색** |
| 구분 / 패널 | **연회색** |
| 포인트 | **파란색** (주요 액션 · 활성 메뉴만) |

포인트 컬러로 화면 전체를 물들이지 않는다.

### 형태

- **둥근 카드** — 얇은 보더, 과한 그림자 없음
- 둥근 버튼 — 주요 CTA는 pill에 가깝게도 가능
- 입력 · 모달도 같은 radius 계열로 통일

### 여백

- **깔끔한 여백** — 답답한 밀집 레이아웃 금지
- 섹션 간격 · 카드 패딩을 넉넉히
- iPad 터치에 충분한 크기 (대략 최소 44px)

### 톤

- Notion + Obsidian + Linear 느낌
- 장식용 배지 · 네온 · 다층 글로우 지양
- 한 화면에 역할을 너무 많이 넣지 않음

---

## 레이아웃

작품 선택 후 작업실:

```
┌──────────┬─────────────────────┐
│ Sidebar  │ Header (작품 제목)   │
│          ├─────────────────────┤
│          │ Content             │
└──────────┴─────────────────────┘
```

- Header에는 **현재 작품 제목** 위주
- 첫 화면(작품 목록)에는 작업실 사이드바를 쓰지 않음
