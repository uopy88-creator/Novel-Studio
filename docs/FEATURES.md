# Novel Studio Features

이 문서는 **Novel Studio가 무엇을 위한 제품인지**,  
그리고 **어떤 기능을 목표로 하는지**를 설명한다.

---

## 목적

Novel Studio는 **「소설만을 위한 프로그램」이 아니다.**

> **창작 작업을 위한 작업실**

이다.

즉,

- 워드프로세서처럼 “글자만 치는 도구”도 아니고
- 세계관 위키처럼 “설정만 쌓는 도구”도 아니다

**작품을 열고 → 문서를 정리하고 → 원고를 쓰고 → 인물·메모·복선을 옆에 두는**  
작업 공간을 만드는 것이 목적이다.

---

## 지원하는 작품 유형

장르를 소설로만 제한하지 않는다.

| 유형 | 예시 |
|------|------|
| 소설 | 장편, 단편, 웹소설 |
| 시 | 시집, 연작시 |
| 에세이 | 수필, 칼럼 모음 |
| 기타 | 희곡 초고, 연재 기획 등 |

UI와 데이터는 “소설 전용 용어”에 과도하게 묶지 않고,  
**작품(Project) → 문서(Document) → 원고(Manuscript)** 흐름을 공통 뼈대로 둔다.

---

## 핵심 기능

### 1. 작품 관리 (Project)

- 작품 생성 / 수정 / 삭제 / 목록
- 작품마다 독립된 작업실
- 첫 화면은 작품 목록

### 2. 문서 관리 (Document)

- 작품 안의 문서(장/편/섹션) 목록
- 생성 / 수정 / 삭제 / 순서 변경
- 제목, 번호, 짧은 설명, 생성·수정일

> 현재 코드에서는 Document 역할을 `Chapter` 엔티티가 담당한다.  
> 로드맵·제품 언어에서는 **Document**로 부른다.

### 3. 원고 작성 (Manuscript)

- Document에 연결된 본문 작성
- 목록(Document)과 본문(Manuscript)을 분리해 유지보수성을 높인다

### 4. 분량 지표

집필 중 바로 확인할 수 있어야 하는 숫자들이다.

| 지표 | 설명 |
|------|------|
| **글자수** | 공백 포함 글자 수 |
| **공백 제외 글자수** | 공백을 뺀 글자 수 |
| **원고지 분량** | 한국 원고지 기준 환산 |
| **예상 책 페이지** | 대략적인 책 페이지 추정 |

정확한 공식은 구현 단계에서 `Settings` / 통계 모듈에 고정한다.

### 5. 캐릭터 (Characters)

- 인물 카드 (이름, 역할, 짧은 요약, 노트)
- 집필 중 바로 열어보는 용도 (두꺼운 세계관 시트보다 가볍게)

### 6. 대사 보관함 (Dialogue Vault)

- 나중에 쓸 대사, 말투 샘플, 인상적인 한 줄 보관
- 인물·문서에 **느슨하게** 연결 가능

### 7. 복선 (Foreshadowing)

- 심어 둔 것 / 회수할 것 추적
- 문서(장)와 연결해 “어디에 심었는지”를 놓치지 않게 한다

### 8. 메모 (Memo)

- 아이디어, 할 일, 미결 질문
- 문서·인물·복선에 선택적으로 붙일 수 있다

---

## 제품이 아닌 것 (의도적으로)

초기에 헷갈리기 쉬운 범위를 분명히 한다.

| 하지 않는 것 | 이유 |
|--------------|------|
| 만능 세계관 위키 | 집필 흐름보다 설정 수집이 커지기 쉬움 |
| 처음부터 클라우드 필수 | v0.1은 로컬로 검증 |
| 과도한 대시보드 위젯 | 첫 화면은 작품·작업에 집중 |
